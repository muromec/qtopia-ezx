dep()

Note that idep() is generally preferred to dep() because it also applies to indirectly-dependant projects. This is especially important when setting variables like LIBS, INCLUDEPATH and DEFINES.

Usage

    dep(LIBS+=-l$$TARGET)
    dep(DEFINES+=SOME_DEFINE,DEFINES)
    dep(DEFINES+=CALLER=\$$TARGET)

Description

Register a command that dependant projects will run. For example, a library can use this to affect the link line of dependant projects. You can pass a second argument that causes the command to run (and the variable to be exported) so that you can use the same command for both this project and dependant projects.

There are some caveats to dep():

  • The .pro file is processed using the qmake function $$fromfile().
  • Only the DEP_CMDS variable is sent to the calling context.
  • Processing stops at your .pro file with $$fromfile() (ie. none of the post-processing occurs).
  • You can use \$$varname to have your variable evaluated in the calling context.

See also Functions (build system), Dependencies, idep(), and depends().
